🚨 Severity: HIGH
💡 Vulnerability: The CORS origin sanitization logic parsed `VIBEGRAPH_CORS_ORIGINS` but failed to enforce strict hostname verification against a separate trusted domains whitelist (`VIBEGRAPH_ALLOWED_CORS_DOMAINS`). This could allow an attacker to bypass credentials rules if the environment variable was maliciously or accidentally misconfigured with untrusted domains.
🎯 Impact: Attackers could potentially bypass CORS policies and send cross-origin credentialed requests to the backend if the domains list is compromised.
🔧 Fix: Implemented strict hostname validation using `urllib.parse.urlparse` to verify each non-wildcard origin's hostname strictly against the authorized whitelist domains (`localhost,vibegraph.vercel.app`) before appending to `validated_origins`.
✅ Verification: Backend test suite passes, frontend test suite passes, and the manual test cases verified proper rejection of malicious domains (e.g. `http://malicious.com`).
