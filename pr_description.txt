🎯 **What:** This PR fixes a Zip Slip Path Traversal vulnerability by rejecting symlink members inside uploaded zip archives.
⚠️ **Risk:** Without this check, an attacker could upload a ZIP file containing a symlink that resolves to a critical system file (like `/etc/passwd`), followed by a regular file that overwrites it, potentially leading to Remote Code Execution (RCE) or complete system compromise.
🛡️ **Solution:** During zip extraction in `app/routers/upload.py`, we now check the external attributes of each member using `stat.S_ISLNK(member.external_attr >> 16)`. If a symlink is detected, a `ValueError` is raised which results in a 400 Bad Request ("Unsafe zip file detected"). A test was added to `tests/test_upload_security.py` to ensure symlinks are correctly rejected.
