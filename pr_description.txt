💡 **What**: Replaced an O(N) loop iterating over all nodes with an O(1) `Set.size` lookup in `useGhostRunner.js` to calculate `visitedCount`.

🎯 **Why**: In `useGhostRunner.js`, `visitedCount` was recalculated via an `useMemo` block that iterated over all nodes (O(N) operation) on every tick of the high-frequency simulation loop (`stepCount`). Since only navigable nodes are added to `visitedSetRef` during the simulation, iterating the full `nodes` array is redundant and adds unnecessary CPU overhead.

📊 **Impact**: Reduces CPU load and eliminates array traversal during the high-frequency animation loop, significantly dropping execution time from O(N) to O(1) per tick, improving the frame rate and responsiveness of the Ghost Runner simulation on large codebases.

🔬 **Measurement**: Ran `pnpm test` successfully with zero regressions across 172 tests. The optimization maintains exact behavior while removing the O(N) iteration block in `useGhostRunner.js`.
