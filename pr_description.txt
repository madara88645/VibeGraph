💡 What:
Replaced `Object.entries().map().join()` with imperative `for...in` loops in `FileSidebar.jsx` and `ChatDrawer.jsx` when building summary strings.

🎯 Why:
To prevent severe garbage collection pressure and CPU overhead caused by the hidden allocation of multiple intermediate arrays during frequent render cycles or large list traversals.

📊 Impact:
Eliminates O(N) array allocation overhead and multiple iteration passes over object entries entirely, providing a best-case zero-allocation path when deriving string-based properties.

🔬 Measurement:
Profile a workspace with numerous node files or trigger rapid simulation ticks; memory snapshots will show fewer short-lived array allocations.
